
/**
 * Http 超文本传输协议 请求/响应式协议，即客户端与服务器建立链接后，客户端发送请求，服务期收到请求后给与客户端响应
 */

1、Http请求报文构成
    请求行 + 请求头 + 空格 + 请求体

    1) 请求行由(get 2m   post 4g) 请求方法、url、 http协议三部分构成
    2) 请求头由host、connection等组成

2、Http响应报文构成
    Http状态行 + 响应头部 + 空格 + 响应体
    1)Http状态行只http status
        1xx => 服务器接受了请求
        2xx => 服务器接受了请求并成功处理
        3xx => 服务器告诉客户端要重定向   301 永久重定向 302 临时重定向
        4xx => 表示客户端的请求有非法内容   400 客户端请求有语法错误  401指请求未授权  403服务器收到请求但是拒绝提供服务 404请求资源不存在
        5xx => 服务器内部错误  502 bad gateway网关错误，无效的http响应  504gateway timeout网关超时


2、Get、Post的区别
    * Get请求参数拼接在Url中，参数可见安全性较差，可传输数据量有限，一般2kb, 单次请求Get会产生1个TCP数据包， 并且Get请求可以被缓存
    * Post请求参数在消息主体中，不可见较安全，传输数据默认4mb，可以通过php.ini文件调整，单次请求会产生2个TCP数据包，请求不可被缓存

3、Http的优缺点
    * 优点：Http无状态，服务器不需要耗费资源去记录Http的状态，可以将更多资源用于对外服务提供
    * 缺点：一些关联性操作时比较麻烦，如登录、添加购物车、下单、结算 服务器不知道请求有关联每次都需要确认
    * 解决：cookie中存有客户端信息，交互过程中带着cookie

4、Http 和 Https的区别
    * Http明文传输，https加入了SSL/TLS协议使得报文加密传输
    * Http建立链接相对简单，在TCP三次握手后即建立链接；Https在三次握手后还要进行SSL/TLS握手才可建立链接
    * 端口区别，Http是80端口，Https是443
    * Https需要向CA(权威证书机构)申请证书来保障服务器是可信的，Http不需要

5、Cookie&Session区别和作用
    * Cookie保存在客户端，请求时带上用于区分用户信息
    * Session保存在服务器端,主要用于记录用户状态
    * Cookie存放在客户端，安全性比session要差


6、SSL协议 (Secure Sockets Layer 安全套接字)
    主要分为两层，
       一层是记录协议层，建立在可靠的传输之上，为上层提供如数据加密、压缩等功能
       另一层是握手协议层，用于身份认证、密钥交换等
