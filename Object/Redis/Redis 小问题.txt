1、线上redis有1亿个key， 从中找出前缀是xxx的key，怎么办？
    keys 'xxx*'
    == 问题：keys扫描的话在线上会造成主线程的阻塞，对象上服务有较大影响
    == solve scan  scan 0 xxx* ，通过scan命令来查找，不阻塞主线程，返回数据中可能存在部分重复，客户端进行去重即可

2、使用redis实现延迟队列
   使用sortedset数据结构，使用时间戳做score， 消息内容作为key，生产者使用zadd生产消息，消费者使用zrangbyscore 轮询

3、Redis热点Key问题
  redis中的某个key的访问流量巨大，导致该分片压力陡增，成为瓶颈(热门商品、话题等等)

  问题发现：监控key的访问qps，有说超过1000需要关注的
  问题解决
    1) 增加本地缓存，本地缓存过期再去查询redis，但是存在数据不一致问题，本地缓存时效
    2) 拆分key， 将一个key拆分成多分，分摊流量(拆分后的多个key位于多个分片上)
    3) 热点key迁移，将热点key迁移至单独的分片上(要是不行呢？再加副本分摊流量干就完了)，这样不至于影响其他业务，还可以定制化开发
